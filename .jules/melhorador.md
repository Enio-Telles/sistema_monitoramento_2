## 2024-05-18 - Optimized File Discovery
**Learning:** Checking for file existence using `glob()` with an exact pattern is highly inefficient compared to constructing the explicit `Path` and using `.is_file()`, resulting in significant performance overhead (~50x difference observed) when executed in loops across directories with multiple files. Also, filtering early with a more specific glob pattern (e.g., `*{cnpj}*.parquet` vs `*.parquet`) drastically reduces the number of iterations required in fallback loops.
**Action:** Replace `glob` calls that search for exact filenames with `Path(dir / filename).is_file()`. Ensure glob patterns are as restrictive as possible (e.g., adding known substrings like IDs/CNPJs) to minimize loop operations when scanning large directories.
